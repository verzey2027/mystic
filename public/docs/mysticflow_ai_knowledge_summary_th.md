# MysticFlow — สรุปศาสตร์และข้อกำหนดที่ AI ใช้อ่านผล

## 1. วัตถุประสงค์ของระบบ
- รวมองค์ความรู้ด้านศาสตร์ทำนายที่ได้รับความนิยมในไทย เพื่อให้ AI สามารถตอบคำถามและให้คำแนะนำได้สอดคล้องกับวัฒนธรรม
- กำหนดโทน “มูละมุน + ฟันธงอย่างมั่นใจ” พร้อมข้อห้ามและแนวทางการสื่อสารเชิงบวก
- สร้างโครงสร้างข้อมูลมาตรฐานสำหรับการเก็บอินพุต การประมวลผล และรูปแบบข้อความผลลัพธ์ของแต่ละศาสตร์

## 2. หลักการด้านจริยธรรมและความปลอดภัย
1. **การใช้เพื่อความเชื่อและการสะท้อนใจ** ไม่ใช่วิทยาศาสตร์หรือหลักฐานทางการแพทย์/การเงิน
2. ห้ามเนื้อหาที่สร้างความกลัวสุดโต่ง ข่มขู่ สาปแช่ง หรือรับประกันผลแน่นอน (โดยเฉพาะเงินและการเสี่ยงลงทุน)
3. ไม่ให้คำแนะนำด้านการแพทย์ กฎหมาย หรือวิธีหลบเลี่ยงกฎหมาย
4. ทุกคำทำนายต้องมี “คำแนะนำที่ลงมือทำได้จริง” อย่างน้อย 1–3 ข้อ
5. **ความเป็นส่วนตัว**: วัน/เวลา/สถานที่เกิด เลขติดต่อ รูปใบหน้า/ฝ่ามือ/ที่อยู่ เป็นข้อมูลอ่อนไหว → เก็บเท่าที่จำเป็น อนุญาตให้ผู้ใช้กรอกแบบประมาณ และแจ้งว่าสามารถปฏิเสธได้

## 3. โครงสร้างคำตอบมาตรฐาน (Message Format)
- **Summary**: สรุปภาพรวม 2–4 บรรทัด
- **Opportunities**: จุดเด่นหรือโอกาส 1–3 bullet
- **Risks**: สิ่งที่ต้องระวัง 1–3 bullet
- **Actions**: คำแนะนำลงมือทำ 1–3 bullet
- **Timeframe**: กรอบเวลา (ถ้าศาสตร์ทำได้)
- **Confidence/Disclaimer**: ระบุระดับความมั่นใจ (low/med/high) และข้อความย้ำว่าผลเป็นแนวทางเชิงสะท้อนใจ
- สำหรับ Tarot เพิ่มรายการไพ่ (position / card / keywords / meaning)
- สำหรับ Feng Shui เพิ่ม Quick Fixes + Deep Changes + เหตุผลเชิงภาษาคน

## 4. รายการศาสตร์ที่ AI รองรับ
| หมายเลข | ศาสตร์ | ใช้กับ | อินพุตหลัก | เอาต์พุต | หมายเหตุสำคัญ |
|---|---|---|---|---|---|
| 1 | **โหราศาสตร์ไทย** | งาน เงิน จังหวะเวลา | วันเกิด (เวลา/สถานที่ถ้ามี) + คำถาม | โครง Summary-Opportunities-Risks-Actions-Timeframe | ถ้าไม่รู้เวลาเกิดให้ลดความมั่นใจ/อธิบายข้อจำกัด |
| 2 | **ไพ่ยิปซี/ไพ่ทาโรต์** | งาน การตัดสินใจ ความรัก | คำถาม กรอบเวลา Spread | สรุป + รายการไพ่ + คำแนะนำ | หลีกเลี่ยงการสรุปจิตใจคนอื่นแบบ 100% |
| 3 | **เลขศาสตร์** | งาน เงิน ลูกค้า | เลขที่ใช้ (เบอร์/ชื่อ/บ้าน) + เป้าหมาย | โครง Summary-Opportunities-Risks-Actions-Timeframe | เน้นว่าเป็นตัวช่วยเสริม ไม่รับประกันความรวย |
| 4 | **ฮวงจุ้ย** | Productivity เงินเข้า | ประเภทสถานที่ + ปัญหา + (ผัง/ทิศถ้ามี) | Summary + Quick Fixes + Deep Changes + Disclaimer | เริ่มจากการแก้ปัญหาแบบงบต่ำก่อน |
| 5 | **ดูดวงไพ่ทาโรต์ (เชิงลึกจาก MysticFlow)** | Tarot 78 ใบ (Major + Minor) | ไพ่ที่สุ่มได้ + คำถาม | คำทำนาย 3–5 ย่อหน้า + คำแนะนำ | ใช้สเปรด 1/3/5/10 ใบ หรือ Celtic Cross |
| 6 | **ไพ่จิตวิญญาณ / Life Path** | คำแนะนำส่วนบุคคล | วันเกิด → Life Path Number | ไพ่ประจำตัว + ความหมาย + คำแนะนำเฉพาะบุคคล | รองรับ Master Numbers 11/22 |
| 7 | **ดวงชะตา (Western Horoscope)** | ภาพรวมชีวิต รัก งาน | วันเกิด + ราศี | ตารางราศี + ธาตุ + แนวโน้ม | ระบุธาตุ 4 ธาตุเพื่ออ่านความเข้ากัน |
| 8 | **ปีนักษัตรจีน + ธาตุ 5** | บุคลิก ทีมเวิร์ก | ปีเกิด | สไตล์ทำงาน + คู่ทีมที่ส่งเสริม + ข้อควรระวัง | ใช้ Wu Xing (ไม้ ไฟ ดิน โลหะ น้ำ) |
| 9 | **ความเข้ากัน (Compatibility)** | รัก มิตรภาพ งาน | ข้อมูลคู่เทียบ (ราศี/ธาตุ/เลข) | คะแนน 0–100 + ระดับ + เคล็ดลับปรับตัว | ไฟ+ลม, ดิน+น้ำ ส่งเสริมกัน; ไฟ+น้ำ ขัดแย้ง เป็นต้น |
| 10 | **ดูดวงสายงาน (Work Focus Pack)** | งาน/อาชีพ/การเงิน | สถานะการงาน สายอาชีพ กรอบเวลา ข้อจำกัด | Summary → Opportunities → Risks → Actions → Timeframe | ปิดท้ายด้วยแผนปฏิบัติ 1–3 ขั้นตอน |
| 11 | **ดูลายมือ** | บุคลิก/ความอึด | รูปฝ่ามือหรือคำบรรยายเส้นสำคัญ | จุดเด่น จุดที่ต้องระวัง คำแนะนำพักผ่อน | ห้ามผูกกับการแพทย์หรือโรคร้าย |
| 12 | **โหงวเฮ้ง** | ภาพลักษณ์การทำงาน | รูปหน้า/คำบรรยาย (ไม่บังคับ) | คำแนะนำการนำเสนอ/สื่อสาร | หลีกเลี่ยงเหมารวม/เหยียด |
| 13 | **ปฏิทินจันทรคติ + ฤกษ์ยาม** | เลือกวันมงคล | กิจกรรม + ช่วงวันที่สะดวก + ข้อจำกัด | 2–3 ช่วงเวลาพร้อมเหตุผล + แผนสำรอง | ระบุเหตุผลภาษาคน (เช่น วันธงชัย/วันอธิบดี) |

> เสริม: สีมงคลประจำวัน, ทำนายฝัน, Oracle Cards, ตัวอย่างคำทำนายสำหรับงาน/ความรัก/ความสัมพันธ์ (ไว้ใน Prompt Pack และ Example)

## 5. ข้อกำหนดเชิงข้อมูล (Dataset Schema)
- **Metadata**: `kb_version = 1.0`, `locale = th-TH`, `domain = divination_thailand`
- **Tone**: `style = มูละมุน`, `delivery = ฟันธงตรงๆ`, ต้องมี `actionable_advice`, หลีกเลี่ยง `fearmongering`, `fatalism`, `guarantees`
- **Global Safety**: ตรงกับข้อ 2 พร้อม `hard_limits` และ `privacy.sensitive_inputs`
- **Message Format**:
  - Work Reading JSON: `summary`, `opportunities[]`, `risks[]`, `actions[]`, `timeframe`, `confidence`, `disclaimer`
  - Tarot Output: เพิ่ม `cards[] {position, card, keywords[], meaning}`
  - Feng Shui Output: `quick_fixes[]`, `deeper_changes[]`
- **System Cards**: แต่ละ `system_id` ระบุ `primary_use`, `required_inputs` (ชนิดข้อมูล, ความจำเป็น, enum/notes) และ `output` schema
- **Example Records**: ให้ตัวอย่างการ map อินพุตจริง → เอาต์พุตตาม schema (เช่น Tarot 3 ใบสำหรับการย้ายงาน)

## 6. ขั้นตอนและ Best Practices สำหรับทีม AI
1. **เก็บข้อมูลอินพุต** ผ่านฟอร์มที่แยกตามศาสตร์ พร้อม validation (เช่น เวลาเกิดไม่จำเป็น → ปรับความมั่นใจ)
2. **เลือก Template/Persona** ให้ตรงบริบท: Work Reading, Love Reading, Matching, Career Matching ฯลฯ
3. **Pre-processing**: สุ่มหรือจัดไพ่/ชุดข้อมูลตามกฎ (เช่น สเปรดไพ่, Life Path Number, คะแนนเลขศาสตร์)
4. **Inference**: ป้อนข้อมูล + prompt โครงสร้างเพื่อให้ LLM เติมเนื้อหาเชิงมนุษย์ พร้อมน้ำเสียงเฉพาะ
5. **Post-processing & Safety Check**:
   - ตรวจคำต้องห้าม/โทนเชิงลบสุดโต่ง
   - เติม disclaimer และระดับความมั่นใจ
   - ให้คำแนะนำ actionable ครบถ้วน
6. **Maintenance**: ทบทวนความถูกต้องทุก 3 เดือน, เก็บ feedback ผู้ใช้, เพิ่มศาสตร์ใหม่ตามความนิยม, อัปเดต tone/ตัวอย่างให้ทันสมัย

## 7. สรุปสำหรับทีมใช้งาน
- MysticFlow ใช้องค์ความรู้ 10+ ศาสตร์ โดยมี core 4: โหราศาสตร์ไทย, ไพ่ทาโรต์, เลขศาสตร์, ฮวงจุ้ย
- ทุกศาสตร์ถูกควบคุมด้วย schema ร่วม (summary → actions → timeframe) เพื่อให้ AI ตอบได้เป็นระบบ
- ความปลอดภัยและความเป็นส่วนตัวคือเงื่อนไขหลัก ต้องแจ้งผู้ใช้ทุกครั้งว่านี่คือคำแนะนำเชิงวัฒนธรรม/ความเชื่อ และตัดสินใจด้วยตนเอง
- ทีม Dev/PM ต้องเตรียมโมดูลตาม `system_id`, บังคับ JSON format, ใส่ safety layer (คำต้องห้าม + การ redirect เมื่อผู้ใช้เครียด)
